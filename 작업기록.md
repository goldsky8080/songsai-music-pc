- 다음 세션에서 바로 이어받을 수 있도록 결과, 메모, 다음 할 일을 남긴다.

## 2026-04-11

### 한 일

- 신규 Ubuntu 서버 기본 하드웨어와 네트워크 상태를 확인했다.
- 500GB HDD를 초기화해 `/srv/archive`로 마운트했다.
- Docker와 Docker Compose를 설치했다.
- `music`, `wrapper`, `bank-bridge` 저장소를 신규 서버에 맞췄다.
- `songsai-bank-bridge` 로컬 폴더를 git 저장소로 만들고 GitHub에 올렸다.
- `D:\music`의 문서 정리 커밋을 `songsai-music`에 푸시했다.
- 로컬 `wrapper` 상태를 기준으로 `songsai-api`를 강제 동기화했다.
- PostgreSQL 컨테이너를 올리고 다음 DB를 만들었다.
  - `songsai_music`
  - `songsai_bank_bridge`
- `music`과 `bank-bridge`에 Prisma migration을 적용했다.
- `music`, `bank-bridge`, `wrapper`의 기본 실행을 확인했다.

### 메모

- `music`은 `3000`에서 응답 확인 완료
- `bank-bridge`는 `3301`에서 응답 확인 완료
- `wrapper`는 테스트 중 `3002`에서 떴고, 고정 포트 정리가 필요하다
- PostgreSQL 비밀번호는 임시값이므로 나중에 더 강하게 바꾸는 것이 좋다
- 로컬 `D:\music`에는 문서 커밋과 별도로 아직 정리되지 않은 변경이 남아 있다

### 다음 할 일

- 각 서비스 `.env` 값을 실제 운영 기준으로 다시 채운다.
- Nginx와 80/443 연결을 준비한다.
- 신규 서버를 다음 운영 대상 기준으로 정리한다.
