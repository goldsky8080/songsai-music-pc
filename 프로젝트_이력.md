# 프로젝트 이력

이 문서는 예전 handoff, plan, 진행 메모 문서를 git 기준으로 다시 읽어 압축한 이력 문서입니다.

## 큰 흐름

- 2026-03-24: 메인 서비스 초기 구현
- 2026-03-26: 관리자, 쿠폰, Google 로그인, provider 재정리
- 2026-03-28 ~ 2026-03-30: 입금 요청, 다운로드, 비디오, aligned lyrics 확장
- 2026-04-01 전후: `MusicAsset` 기반 자산 구조 강화
- 2026-04-10 ~ 2026-04-11: 문서 정리, 3개 저장소 관계 재정리, 신규 서버 세팅

## 1. 초기 구현 단계

### 2026-03-24

초기 메인 서비스의 골격이 들어간 시기입니다.

핵심:

- 인증 흐름
- 사용자/크레딧 관련 모델
- 음악 생성 요청
- 최근 곡 목록
- 다운로드 흐름

이 시점의 성격은 “기본적인 음악 생성 서비스”에 가까웠습니다.

## 2. 제품 기능 확장 단계

### 2026-03-26

운영 가능한 서비스 쪽으로 범위가 확장됐습니다.

추가된 흐름:

- 관리자 기능
- 쿠폰
- Google 로그인
- 보너스 트랙
- 입금 요청

같은 시기 provider 명칭과 환경변수 구조도 `SONGS_*` 방향으로 정리되었습니다.

## 3. 운영 자동화와 결제 흐름 확장

### 2026-03-28 ~ 2026-03-30

이 구간은 생성 기능보다 운영 보완이 많이 들어간 시기입니다.

주요 흐름:

- 입금 요청 UX 강화
- 입금 자동 매칭 보조 흐름
- 다운로드 안정화
- aligned lyrics 확장
- 비디오 생성 관련 흐름 정리
- OAuth 및 wrapper 연동 보완

## 4. 자산 구조 강화

### 2026-04-01 전후

이 시기에는 `MusicAsset` 기반 자산 관리가 더 중요해졌습니다.

핵심 방향:

- mp3
- 이미지
- 가사
- 비디오

를 단순 결과 파일이 아니라 재사용 가능한 자산으로 다루기 시작했습니다.

## 5. 문서와 구조 재정리

### 2026-04-10 ~ 2026-04-11

이 시기에는 문서 중복을 줄이고, 프로젝트를 다시 읽을 수 있게 정리하는 작업이 진행됐습니다.

주요 내용:

- 중복 문서 삭제
- 핵심 문서 최소화
- 3개 저장소를 한 시스템으로 다시 파악
- 신규 서버 세팅
- 신규 서버에서 DB, migration, 실행 확인
- 장기적으로 `songsai-api` 중심 구조로 옮기는 방향 정리

## 현재 시점의 판단

지금까지의 흐름을 보면, 프로젝트는 다음 단계로 가는 것이 맞습니다.

- `music`은 점차 얇은 프론트
- `songsai-api`는 점차 메인 백엔드
- `songsai-bank-bridge`는 specialized helper

즉 이력상으로도 “메인 앱에 많은 책임이 쌓인 구조”에서 “API 중심 구조”로 옮겨가는 것이 자연스러운 다음 단계입니다.
