# 운영·배포

이 문서는 현재 운영과 배포 관점에서 중요한 사항을 정리합니다.

## 현재 확인된 신규 서버 기준

- Ubuntu 서버
- Docker 설치 완료
- PostgreSQL Docker 컨테이너 구성 완료
- `/srv/archive` 아카이브 디스크 구성 완료
- `music`, `wrapper`, `bank-bridge` 코드 배치 및 기본 실행 확인 완료

## 현재 포트 상황

테스트 기준:

- `music`: `3000`
- `bank-bridge`: `3301`
- `wrapper`: 테스트 시 `3002`

권장 고정 포트:

- `music`: `3000`
- `wrapper`: `3101`
- `bank-bridge`: `3301`

## 데이터베이스

현재 DB:

- `songsai_music`
- `songsai_bank_bridge`

PostgreSQL은 Docker로 관리 중입니다.

## 운영 시 원칙

1. 코드 수정은 서버보다 로컬에서 먼저 한다.
2. 배포는 git push / server pull 흐름을 기본으로 한다.
3. `.env`는 서버별로 따로 관리한다.
4. 생성 산출물은 코드 저장소와 분리한다.
5. N100은 legacy 운영 기준, 신규 서버는 다음 운영 대상 기준으로 본다.

## 지금 남아 있는 운영 과제

- `wrapper` 고정 포트 정리
- 각 서비스 `.env` 실제 운영값 정리
- Nginx 구성
- 80/443 연결
- HTTPS 적용
- 백업과 아카이브 흐름 정리

## 장기 권장 방향

- 외부 공개는 메인 앱 중심
- `wrapper`, `bank-bridge`는 내부 서비스 또는 제한 공개 기준
- 결제는 PayPal 중심 + bank transfer 보조 구조
