# PC 버전 프로젝트 구상

이 문서는 앞으로 만들 SONGSAI PC 전용 프론트 프로젝트의 초안 문서입니다.

목적은 다음과 같습니다.

- 모바일 중심 `songsai-music`과 별도로 PC 전용 제품 방향을 정한다
- 디자인 무드와 실제 정보 구조를 분리해서 판단한다
- 새 세션에서도 바로 PC 프로젝트 방향을 이해할 수 있게 한다

## 핵심 결론

PC 버전은 기존 `D:\music` 안에서 반응형으로 억지로 키우기보다, 별도 프로젝트로 분리하는 것이 좋습니다.

권장 방향:

- `D:\music`
  - 모바일 중심 또는 모바일 우선 프론트
- 새 PC 프로젝트
  - PC 전용 프론트
- `songsai-api`
  - 모바일과 PC가 같이 쓰는 공용 백엔드

## 왜 PC 프로젝트를 따로 가는가

이유는 명확합니다.

- 모바일과 PC는 정보 밀도와 작업 흐름이 다르다
- PC 쪽은 더 화려하고 풍부한 UI가 필요하다
- 생성, 자산, 비디오, 결제, 작업 상태를 한 화면에서 더 많이 보여줄 수 있다
- 모바일을 기준으로 만든 구조 위에 PC UI를 얹으면 다시 책임이 꼬일 가능성이 높다

즉 PC는 단순 확대판이 아니라 별도 경험으로 설계하는 것이 맞습니다.

## 디자인 참고 샘플에 대한 판단

참고한 샘플:

- `D:\one-music-gh-pages`

판단:

- 무드 참고용으로는 좋다
- 구조 참고용으로는 부족하다
- 실제 코드 기반으로 그대로 재사용하는 것은 비추천이다

좋은 점:

- 음악 서비스다운 강한 비주얼 무드
- 큰 히어로 영역
- 대형 카드와 슬라이드 리듬
- “음악 제품처럼 보이는” 첫인상

안 맞는 점:

- 오래된 정적 HTML 템플릿 구조
- 앨범/홍보 사이트 구조 중심
- 생성 앱, 자산 패널, 크레딧, 비디오 상태 같은 앱형 정보 구조에는 맞지 않음

결론:

- 샘플의 분위기와 질감은 참고한다
- 실제 정보 구조와 코드 구조는 새로 만든다

## PC 버전 제품 방향

PC 버전은 다음 인상을 목표로 합니다.

- 다이내믹함
- 프리미엄 느낌
- 스튜디오 같은 분위기
- “AI 음악 작업 공간” 같은 몰입감

즉 일반적인 SaaS 대시보드보다 더 강한 시각적 방향이 있어야 합니다.

## PC 메인 화면 초안

### 1. 히어로 + 현재 상태 요약

첫 화면 상단은 강한 비주얼로 시작합니다.

구성:

- 브랜드 헤드라인
- 현재 크레딧
- 오늘 생성 수
- 처리 중 작업 수
- 최근 완성 곡 미리보기
- 즉시 생성 버튼

이 영역은 `one-music-gh-pages`의 대형 히어로 감성을 참고해도 좋습니다.

### 2. 생성 콘솔

PC 버전의 중심입니다.

구성:

- 제목 입력
- 프롬프트
- 스타일 입력
- 가사 모드 선택
- MR 여부
- 생성 버튼
- 예상 크레딧 소모

이 영역은 한눈에 보고 바로 작업할 수 있게 커야 합니다.

### 3. 최근 곡 / 현재 작업 패널

구성:

- 생성 중인 작업 리스트
- 최근 완성 곡
- 실패한 작업 요약
- 각 항목별 빠른 액션
  - 재생
  - 다운로드
  - 비디오 만들기
  - 상세 보기

### 4. 자산 패널

구성:

- MP3
- 커버 이미지
- 가사
- 비디오
- 다운로드 가능 여부

PC에서는 이 자산 패널을 별도 영역으로 크게 보여주는 것이 좋습니다.

### 5. 결제 / 충전 패널

구성:

- 현재 잔액
- 빠른 충전 버튼
- PayPal 결제 진입
- 무통장 입금 보조 진입

### 6. 운영 상태 패널

선택 영역이지만 PC 버전에서는 유용합니다.

구성:

- API 상태
- wrapper 상태
- 비디오 처리 상태
- 최근 오류 요약

## 화면 구조 초안

대략 이런 레이아웃이 좋습니다.

```text
상단: 강한 히어로 + 잔액/상태 요약

좌측 메인:
- 생성 콘솔
- 최근 곡/처리중 작업

우측 사이드:
- 자산 패널
- 충전 패널
- 상태 패널
```

## 모바일과의 차이

### 모바일

- 빠르고 얇게
- 생성 한 번, 결과 한 번에 초점
- 패널 수 최소화

### PC

- 한 화면에 더 많은 정보
- 여러 패널 동시 노출
- 작업 흐름 전체를 한눈에 볼 수 있게
- 더 화려한 타이포그래피와 배경, 모션 허용

## API 사용 방식

PC 프로젝트는 자체적으로 무거운 비즈니스 로직을 가지지 않습니다.

원칙:

- PC 프론트는 `songsai-api`를 호출한다
- 결과를 받아 렌더링한다
- 결제, 생성, 자산, 비디오는 API 중심으로 처리한다

즉 PC 버전은 강한 프론트이지만 얇은 클라이언트로 가는 것이 좋습니다.

## 프로젝트 생성 방향

새 PC 프로젝트는 아래 기준으로 시작하는 것이 좋습니다.

- 새 프론트 프로젝트로 분리
- TypeScript 기반
- `songsai-api` 공용 사용
- 디자인은 별도 테마와 컴포넌트 체계로 설계

권장 이름 예시:

- `songsai-pc`
- `songsai-desktop-web`

## CAPTCHA 관련 메모

PC 프로젝트 자체가 CAPTCHA를 직접 다루지는 않습니다.

하지만 전체 구조상 반드시 기억해야 할 점:

- 실제 provider 측 CAPTCHA 운영은 wrapper 계층에서 해결한다
- Ubuntu 데스크탑에서 보이는 Chromium/Chrome 계열 브라우저를 띄우는 수동 CAPTCHA 흐름이 이미 존재한다
- `manual CAPTCHA lock` 같은 운영 개념은 유지해야 한다

즉 PC 프로젝트는 CAPTCHA를 직접 처리하지 않고, `songsai-api`와 wrapper 운영 흐름을 전제로 설계해야 합니다.

## 첫 구현 우선순위

PC 프로젝트를 실제로 시작할 때 우선순위는 다음이 좋습니다.

1. 메인 레이아웃과 브랜드 방향
2. 히어로 + 상태 요약
3. 생성 콘솔
4. 최근 곡 패널
5. 자산 패널
6. 결제 패널
7. 상태 패널

## 최종 판단

가장 좋은 방향은:

- 모바일과 PC를 분리한다
- `songsai-api`를 공용 백엔드로 둔다
- `one-music-gh-pages`는 디자인 무드 참고용으로만 쓴다
- 실제 구조는 SONGSAI 작업 흐름 기준으로 새로 설계한다

이 문서는 앞으로 PC 프로젝트를 새로 시작할 때 기본 기준 문서로 사용합니다.
