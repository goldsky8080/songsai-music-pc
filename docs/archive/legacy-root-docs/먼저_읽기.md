# 먼저 읽기

새 세션이 시작되면 이 문서를 가장 먼저 읽습니다.

## 읽는 순서

1. 이 문서
2. [작업기록](D:\music\docs\작업기록.md)
3. [재구축 방향](D:\music\docs\재구축_방향.md)
4. 필요하면 아래 문서 추가 확인
   - [개발·배포 작업방식](D:\music\docs\개발_배포_작업방식.md)
   - [PC 버전 프로젝트 구상](D:\music\docs\PC_버전_프로젝트_구상.md)
   - [시스템 구조](D:\music\docs\시스템_구조.md)
   - [운영·배포](D:\music\docs\운영_배포.md)
   - [전체구조 개요](D:\music\docs\전체구조_개요.md)
   - [프로젝트 이력](D:\music\docs\프로젝트_이력.md)

## 세션 시작 체크리스트

1. [작업기록](D:\music\docs\작업기록.md)의 가장 최근 날짜를 읽는다.
2. 현재 무엇이 끝났고 무엇이 남았는지 파악한다.
3. 이번 작업이 주로 어느 저장소인지 확인한다.
   - `D:\music`
   - `D:\songsai-bank-bridge`
   - `D:\wrapper\suno-api`
4. 문서만 믿지 말고, 필요하면 코드와 실행 상태를 직접 확인한다.

## 문서 업데이트 규칙

- [작업기록](D:\music\docs\작업기록.md)은 매번 자동으로 쓰지 않는다.
- 사용자가 마지막에 “오늘 한 것 정리해줘”라고 요청했을 때만 반영한다.
- 구조가 바뀌면 [시스템 구조](D:\music\docs\시스템_구조.md) 또는 [전체구조 개요](D:\music\docs\전체구조_개요.md)를 수정한다.
- 운영 방식이 바뀌면 [운영·배포](D:\music\docs\운영_배포.md)를 수정한다.
- 장기 방향이 바뀌면 [재구축 방향](D:\music\docs\재구축_방향.md)을 수정한다.

## 현재 핵심 인식

- `music`은 메인 사용자 서비스다.
- `wrapper`는 가장 민감한 연동 계층이다.
- `songsai-bank-bridge`는 입금 자동화 보조 서비스다.
- 신규 서버는 다음 구조를 준비하는 서버이고, N100은 현재 legacy 운영 기준이다.
