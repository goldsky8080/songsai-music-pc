# 시스템 구조

이 문서는 현재 코드 기준의 구조와 핵심 개념을 정리합니다.

## 기술 스택

- Next.js
- React
- TypeScript
- PostgreSQL
- Prisma
- ffmpeg 계열 도구

## 현재 메인 서비스 구조

### 사용자 화면

메인 서비스는 `D:\music` 안의 Next.js 앱입니다.

주요 역할:

- 로그인
- 크레딧 조회
- 음악 생성 요청
- 최근 곡 목록 표시
- 다운로드
- 비디오 생성

### 생성 처리

실제 음악 생성은 외부 provider와 wrapper 계층을 통해 처리됩니다.

현재 흐름:

1. 사용자가 메인 서비스에서 생성 요청
2. 메인 서비스가 요청과 크레딧 처리
3. wrapper/provider 계층이 외부 생성 서비스를 호출
4. 결과를 다시 메인 서비스가 저장하고 노출

### 데이터 모델 핵심

현재 코드 기준 핵심 개념:

- `User`
- `CreditGrant`
- `Transaction`
- `Music`
- `MusicAsset`
- `Video`
- `GenerationJob`
- `DepositRequest`
- `Coupon`

## 현재 구조의 한계

- 메인 프론트가 다소 많은 책임을 가지고 있다.
- 생성, 자산, 크레딧, 다운로드, 비디오 관련 로직이 분산되어 있다.
- 장기적으로는 API 중심 구조로 다시 정리하는 것이 더 적절하다.

## 권장 목표 구조

### `songsai-music`

- 사용자 화면 전용
- 모바일 우선
- PC는 더 풍부한 UI
- API 호출 중심

### `songsai-api`

- 메인 비즈니스 로직
- 크레딧
- 생성 요청
- 자산
- 다운로드
- 비디오
- 결제 처리

### `songsai-bank-bridge`

- 은행 입금 자동화 전용

## 현재 기준 결론

현재 구조는 운영을 이어갈 수는 있지만, 앞으로는 `songsai-api` 중심 구조로 점진적으로 재배치하는 것이 좋습니다.
