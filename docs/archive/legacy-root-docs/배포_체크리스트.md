# 배포 체크리스트

## A. songsai-api 배포

### 배포 전 확인

- 로컬에서 기능 확인 완료
- 필요한 환경값 변경 여부 확인
- Google OAuth 주소 변경 필요 여부 확인

### 서버 반영 순서

```bash
cd ~/services/wrapper
git pull origin main
npm install
npm run build
pm2 restart songsai-api --update-env
pm2 save
```

### 배포 후 확인

```bash
curl https://api.songsai.org/api/v1/me
```

비로그인 상태면 아래 응답도 정상이다.

```json
{"error":"Unauthorized"}
```

## B. songsai-music-pc 배포

### 배포 전 확인

- 로컬 프론트에서 UI 확인 완료
- `NEXT_PUBLIC_SONGSAI_API_URL`이 운영값인지 확인
- 로그인/로그아웃/이름 표시 확인

### 서버 반영 순서

```bash
cd ~/services/songsai-music-pc
git pull origin main
npm install
npx prisma generate
npm run build
pm2 restart songsai-music-pc --update-env
pm2 save
```

### nginx / HTTPS

```bash
sudo nginx -t
sudo systemctl reload nginx
sudo certbot --nginx -d pc.songsai.org
```

### 배포 후 확인

- `https://pc.songsai.org` 접속
- 홈 화면 정상 표시
- 로그인 사용자 이름 표시
- `/login` 진입 및 로그아웃 동작

## C. 운영 배포 공통 금지사항

- `.env.local` 값을 서버에 그대로 복사하지 않는다.
- 서버에서 테스트용 임시 env 값을 오래 유지하지 않는다.
- 빌드 실패 상태에서 `pm2 restart`만 하지 않는다.
- 운영 서버에서 직접 코드 수정 후 방치하지 않는다.

## D. 문제 생겼을 때 먼저 볼 것

### API 문제

- `pm2 logs songsai-api`
- `~/services/wrapper/.env`
- `curl https://api.songsai.org/api/v1/me`

### PC 프론트 문제

- `pm2 logs songsai-music-pc`
- `~/services/songsai-music-pc/.env`
- `curl http://127.0.0.1:3000`
- `https://pc.songsai.org`
