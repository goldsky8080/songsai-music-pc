# 환경 관리 가이드

## 목표

개발은 로컬에서 빠르게 진행하고, 운영 배포 때는 환경값만 바꿔도 같은 코드가 안정적으로 동작하도록 유지한다.

핵심 원칙:

- 코드와 기능 로직은 최대한 공통으로 유지한다.
- 환경별 차이는 `.env` 파일로만 분리한다.
- 민감한 값은 저장소에 올리지 않는다.
- 저장소에는 항상 예시 파일만 남긴다.

## 저장소별 역할

### 1. songsai-music-pc

- 역할: PC 웹 프론트
- 로컬 개발 주소: `http://localhost:3000`
- 운영 주소: `https://pc.songsai.org`

### 2. songsai-api

- 역할: 공용 인증 및 API 백엔드
- 로컬 개발 주소: `http://localhost:3100`
- 운영 주소: `https://api.songsai.org`

## 절대 원칙

- `.env`, `.env.local`은 Git에 올리지 않는다.
- 저장소에는 `.env.example`, `.env.production.example`만 유지한다.
- 로컬 개발 중 운영 `.env` 값을 직접 덮어쓰지 않는다.
- 운영 서버에서 환경값을 바꾼 뒤에는 반드시 `pm2 restart --update-env`를 실행한다.

## 권장 파일 구성

### songsai-music-pc

- `.env.example`
- `.env.production.example`
- `.env.local` : 로컬 전용
- 서버 `.env` : 운영 전용

### songsai-api

- `.env.example`
- 로컬 `.env`
- 서버 `.env`

## 가장 자주 바뀌는 값

- 프론트의 API 주소
- 백엔드의 프론트 허용 주소
- Google OAuth 관련 주소
- 데이터베이스 접속 문자열

이 4가지는 배포 전에 항상 다시 확인한다.
