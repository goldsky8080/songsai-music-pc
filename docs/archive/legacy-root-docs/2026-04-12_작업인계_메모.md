### 2. songsai-api 음악 생성/조회 흐름

- `POST /api/v1/music`
  - Music row 생성
  - GenerationJob(`MUSIC_GENERATION`) 생성
- `GET /api/v1/music`
  - 내 음악 목록
- `GET /api/v1/music/recent`
  - 완료곡 최신 목록
- `POST /api/v1/jobs/run`
  - 수동 worker 실행

관련 파일:
- `D:\wrapper\suno-api\src\app\api\v1\music\route.ts`
- `D:\wrapper\suno-api\src\app\api\v1\music\recent\route.ts`
- `D:\wrapper\suno-api\src\app\api\v1\jobs\run\route.ts`
- `D:\wrapper\suno-api\src\server\music\worker.ts`
- `D:\wrapper\suno-api\src\server\music\provider.ts`

### 3. GenerationJob / worker 흐름

- generation job와 poll job 분리
- 생성 직후에는 queued/submitted 상태
- 5분 후 poll job 실행 시 completed 반영 가능
- 로컬에서 실제로
  - generation worker 실행
  - 5분 뒤 poll worker 실행
  - recent/completed 반영
까지 확인함

## 주의사항

- `npm run build`는 dev 서버가 `.next\trace`를 잡고 있으면 `EPERM` 날 수 있음
- Windows PowerShell에서 한글 로그가 깨져 보여도 실제 기능과 무관한 경우가 많음
- `pnpm` 경고, `punycode` 경고는 현재 우선순위 낮음
- songsai-api 쪽은 enum/db push 상태를 항상 맞춰야 함
