# 재부팅 후 서버 점검 체크리스트

신규 서버(`songs-X99`)를 재부팅한 뒤, 서비스가 정상 복구되었는지 확인하는 순서입니다.

이 문서는 아래 4가지를 빠르게 확인하는 목적입니다.

1. 서버 자체가 정상 부팅됐는지
2. `pm2` 앱(`songsai-api`, `songsai-music-pc`)이 자동 복구됐는지
3. `nginx`가 정상인지
4. 외부 도메인(`api.songsai.org`, `pc.songsai.org`)이 응답하는지

---

## 1. 서버 접속

먼저 서버에 SSH로 접속합니다.

```bash
ssh -p 2233 songs@125.189.206.134
```

설명:
- 서버가 실제로 켜졌는지 확인하는 첫 단계입니다.
- 접속이 안 되면 서버 부팅, 공유기 포트포워딩, 공인 IP, 네트워크 상태를 먼저 확인해야 합니다.

---

## 2. 내부 IP / 공인 IP 확인

```bash
hostname -I
curl ifconfig.me
```

설명:
- `hostname -I`
  서버가 현재 어떤 내부 IP를 받았는지 확인합니다.
- `curl ifconfig.me`
  외부에서 보이는 공인 IP를 확인합니다.

점검 포인트:
- 내부 IP가 예전과 같아야 공유기 포트포워딩이 그대로 동작할 가능성이 높습니다.
- 우리가 이전에 사용하던 내부 IP는 `192.168.0.6` 이었습니다.

---

## 3. PM2 앱 상태 확인

```bash
pm2 status
```

설명:
- `songsai-api`
- `songsai-music-pc`

이 두 개가 `online`인지 확인합니다.

정상 예시:

```text
songsai-api        online
songsai-music-pc   online
```

비정상일 때:
- 앱이 없거나 `stopped`, `errored` 상태면 아래 복구 명령을 사용합니다.

복구:

```bash
pm2 resurrect
pm2 status
```

설명:
- 재부팅 후 `pm2` 저장 목록(`dump.pm2`) 기준으로 앱을 다시 올립니다.

---

## 4. nginx 상태 확인

```bash
sudo systemctl status nginx
```

설명:
- 웹 서버/리버스 프록시 역할을 하는 `nginx`가 살아 있는지 확인합니다.

정상 포인트:
- `active (running)` 으로 보여야 합니다.

비정상일 때:

```bash
sudo systemctl daemon-reload
sudo systemctl restart nginx
sudo systemctl status nginx
```

설명:
- 설정 파일 변경이 반영되지 않았거나, nginx가 부팅 후 정상 재시작되지 않았을 때 다시 읽고 재시작합니다.

---

## 5. 로컬 포트 응답 확인

```bash
curl -I http://127.0.0.1:3100
curl -I http://127.0.0.1:3000
```

설명:
- `3100`은 `songsai-api`
- `3000`은 `songsai-music-pc`

즉 앱 자체가 서버 안에서 응답하는지 확인하는 단계입니다.

정상 포인트:
- 둘 다 `HTTP/1.1 200 OK` 또는 정상적인 Next.js 응답 헤더가 보여야 합니다.

비정상일 때:
- `Connection refused` 이면 해당 앱이 실제로 안 떠 있는 상태입니다.
- 다시 `pm2 status`, `pm2 logs`를 확인합니다.

---

## 6. 외부 도메인 응답 확인

```bash
curl -I https://api.songsai.org
curl -I https://pc.songsai.org
```

설명:
- Cloudflare, DNS, SSL, nginx, 앱까지 전부 이어졌는지 확인하는 단계입니다.

정상 포인트:
- `HTTP/2 200`
- 또는 정상적인 리다이렉트/응답 헤더

비정상일 때:
- `523` 이면 원본 서버 도달 문제
- `522` 이면 연결 타임아웃
- `526` 이면 SSL 인증서 문제

이 경우:
1. `pm2 status`
2. `sudo systemctl status nginx`
3. 공유기 포트포워딩
4. Cloudflare DNS
순서로 다시 확인합니다.

---

## 7. 자동복구 설정이 살아 있는지 확인

```bash
systemctl status pm2-songs
```

설명:
- 재부팅 시 `pm2 resurrect`를 자동으로 해주는 systemd 서비스입니다.

정상 포인트:
- `enabled`
- `active` 또는 최소한 등록된 상태

추가 확인:

```bash
pm2 save
```

설명:
- 현재 정상 상태의 앱 목록을 다시 저장해 둡니다.
- 나중에 또 재부팅될 때 이 상태로 복구됩니다.

---

## 8. 로그 확인이 필요할 때

API 로그:

```bash
pm2 logs songsai-api
```

프론트 로그:

```bash
pm2 logs songsai-music-pc
```

설명:
- 실시간 로그 확인용입니다.
- 나갈 때는 `Ctrl + C` 를 누르면 됩니다.
- 로그 보기만 종료되고 서버는 꺼지지 않습니다.

---

## 빠른 점검용 최소 명령 세트

재부팅 후 빠르게만 확인하려면 이 순서만 써도 됩니다.

```bash
pm2 status
sudo systemctl status nginx
curl -I http://127.0.0.1:3100
curl -I http://127.0.0.1:3000
curl -I https://api.songsai.org
curl -I https://pc.songsai.org
```

---

## 문제 발생 시 복구 기본 순서

```bash
pm2 resurrect
sudo systemctl daemon-reload
sudo systemctl restart nginx
pm2 save
pm2 status
```

설명:
- 앱 복구
- nginx 재반영
- 현재 정상 상태 저장

---

## 최종 정상 기준

아래가 모두 만족되면 정상입니다.

- `pm2 status` 에서 `songsai-api`, `songsai-music-pc` 가 `online`
- `sudo systemctl status nginx` 가 `active (running)`
- `curl -I http://127.0.0.1:3100` 응답 정상
- `curl -I http://127.0.0.1:3000` 응답 정상
- `curl -I https://api.songsai.org` 응답 정상
- `curl -I https://pc.songsai.org` 응답 정상
