# 로컬 / 운영 환경값 정리

## 1. songsai-music-pc

### 로컬 개발

파일: `D:\songsai-music-pc\.env.local`

```env
NEXT_PUBLIC_SONGSAI_API_URL="http://localhost:3100"
```

의미:

- 프론트는 로컬에서 동작
- API도 로컬 `songsai-api`에 붙음
- 가장 편한 개발 모드

### 운영 서버

파일: 서버의 `~/services/songsai-music-pc/.env`

```env
DATABASE_URL="postgresql://songsai:replace-password@localhost:5432/songsai_music_pc?schema=public"
FFMPEG_PATH="ffmpeg"
FFPROBE_PATH="ffprobe"
NEXT_PUBLIC_SONGSAI_API_URL="https://api.songsai.org"
```

의미:

- 프론트는 운영 도메인 `pc.songsai.org`
- API는 운영 도메인 `api.songsai.org`

## 2. songsai-api

### 로컬 개발

파일: `D:\wrapper\suno-api\.env`

```env
DATABASE_URL="postgresql://songsai:replace-password@localhost:5432/music_platform?schema=songsai_api"
APP_URL="http://localhost:3100"
FRONTEND_URL="http://localhost:3000"
FRONTEND_URLS="http://localhost:3000,https://pc.songsai.org"
AUTH_SECRET="local-dev-secret"
GOOGLE_CLIENT_ID="..."
GOOGLE_CLIENT_SECRET="..."
```

### 운영 서버

파일: `~/services/wrapper/.env`

```env
DATABASE_URL="postgresql://songsai:replace-password@localhost:5432/music_platform?schema=songsai_api"
APP_URL="https://api.songsai.org"
FRONTEND_URL="https://pc.songsai.org"
FRONTEND_URLS="https://pc.songsai.org,http://localhost:3000"
AUTH_SECRET="production-secret"
GOOGLE_CLIENT_ID="..."
GOOGLE_CLIENT_SECRET="..."
```

## 3. Google OAuth 기준 주소

### 승인된 JavaScript 원본

- `https://songsai.org`
- `http://localhost:3000`
- `http://localhost:3100`
- `https://pc.songsai.org`

### 승인된 리디렉션 URI

- `https://songsai.org/api/auth/google/callback`
- `http://localhost:3100/api/v1/auth/google/callback`
- `https://api.songsai.org/api/v1/auth/google/callback`

## 4. 추천 개발 방식

평소에는 아래 조합으로 개발한다.

- 프론트: `localhost:3000`
- API: `localhost:3100`

운영 확인이 필요할 때만 아래 조합으로 본다.

- 프론트: `https://pc.songsai.org`
- API: `https://api.songsai.org`
