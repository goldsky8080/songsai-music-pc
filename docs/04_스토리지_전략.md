# 스토리지 전략

## 물리 저장소 역할

### SSD 1TB

- hot storage
- 최근 커버 이미지
- 최근 mp3 캐시
- 렌더 임시 파일
- 최근 생성 비디오

경로:

- `/srv/media/music/...`
- `/srv/media/image/...`
- `/srv/media/video/...`
- `/srv/media/tmp/...`

### NAS 4TB

- archive storage
- 오래된 mp3 / image / video
- 장기 보관

경로:

- `/srv/oldmedia/music/...`
- `/srv/oldmedia/image/...`
- `/srv/oldmedia/video/...`
- `/srv/oldmedia/tmp/...`

### HDD 500GB

- 보조/백업 용도
- 당장 메인 저장소로 사용하지 않음

## 앱이 보는 기준

앱은 물리 디스크를 직접 알지 않는다.

DB에는 다음 메타데이터를 저장한다.

- `storageTier`
- `storageKey`
- `storagePath`
- `publicUrl`
- `sourceUrl`
- `archivedAt`

## 아카이브 정책

- `createdAt`만으로 요청 처리 코드에서 `/srv/media`와 `/srv/oldmedia`를 직접 분기하지 않는다
- 별도 아카이브 작업이 SSD -> NAS 이동과 `storageTier` 갱신을 처리한다

## 현재 구현 원칙

- 커버 이미지는 캐시 우선 후보
- mp3는 다운로드/비디오 렌더 시점 lazy 캐시 허용
- 비디오는 렌더 결과를 우리 저장소에 저장
