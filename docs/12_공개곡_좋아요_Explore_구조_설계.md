# 공개곡 · 좋아요 · Explore 구조 설계

기준일: 2026-04-14

## 목적

SongsAI Music을 개인 작업 툴에서 끝내지 않고, 공개곡 감상과 좋아요, 아티스트 탐색까지 가능한 서비스 구조로 확장한다.

이번 설계의 핵심은 아래 4가지다.

- 생성된 곡은 기본 공개(`public`)로 간다.
- 사용자는 필요한 곡만 개별적으로 비공개(`private`)로 전환한다.
- 로그인하지 않은 사용자도 공개곡 목록을 둘러보고 미리듣기할 수 있다.
- 홈과 전체곡 화면에서 주간/월간 순위를 볼 수 있게 한다.

## 기본 정책

### 1. 공개 여부 기본값

- `Music.isPublic` 기본값은 `true`
- 사용자가 직접 비공개로 바꾼 곡만 `false`

이유:

- 초반에는 공개곡 풀이 빠르게 쌓여야 한다.
- 비로그인 사용자도 둘러볼 곡이 있어야 서비스 매력이 살아난다.
- Explore/랭킹 기능이 실제로 의미를 가지려면 기본 공개가 유리하다.

### 2. 비로그인 사용자 허용 범위

비로그인 사용자도 공개곡에 대해 아래를 허용한다.

- 목록 보기
- 카드 보기
- 미리듣기
- 아티스트 페이지 이동

비허용:

- 다운로드
- 비디오 생성
- 공개 설정 변경
- 좋아요 저장

좋아요는 로그인 사용자만 가능하게 두는 것이 안전하다.

### 3. 타인 곡 접근 범위

타인의 공개곡은 아래만 가능하다.

- 듣기
- 좋아요
- 아티스트 공개곡 보기

타인 곡은 아래가 불가능하다.

- 다운로드
- 비디오 생성
- 자산 상세 접근

### 4. 내 곡 접근 범위

본인 곡은 기존과 동일하게 유지한다.

- 듣기
- 다운로드
- 비디오 생성
- 공개/비공개 전환

## 메뉴 구조 제안

상단 메뉴는 무작정 늘리지 않고 트리 구조로 묶는다.

### 1차 메뉴

- `Home`
- `Studio`
- `Explore`
- `Pricing`
- `Support`

### Studio 하위 메뉴

- `Create`
- `My Assets`

### Explore 하위 메뉴

- `All Songs`
- `Weekly`
- `Monthly`
- `Artists`

## 화면 구조

### 1. Home

홈 화면에는 공개곡 중심 섹션을 추가한다.

- 주간 좋아요 순위
- 월간 좋아요 순위
- 최신 공개곡
- 인기 아티스트

순위 리스트 스타일은 홈 하단 카드형 섹션 느낌으로 구성한다.

권장 UI:

- 좌측: 주간 추천/주간 좋아요 순위
- 가운데: 최근 생성 결과/최신 공개곡
- 우측: 주요 아티스트 또는 인기 활용 장면

### 2. Explore / All Songs

공개곡 전체를 보는 중심 페이지다.

필터/정렬:

- 최신순
- 주간 좋아요순
- 월간 좋아요순

카드 정보:

- 커버
- 제목
- 아티스트 닉네임
- 좋아요 수
- 플레이 버튼

동작:

- 카드 클릭 또는 플레이 버튼으로 미리듣기
- 닉네임 클릭 시 아티스트 페이지 이동
- 다운로드 버튼은 노출하지 않음

### 3. Artist 페이지

닉네임 기준 공개곡 아카이브 페이지

권장 경로 예시:

- `/artists/[handle]`

표시 정보:

- 닉네임
- 공개곡 목록
- 총 공개곡 수
- 총 좋아요 수

### 4. My Assets

기존 자산 관리 화면 유지

추가 고려:

- 각 곡에 `공개/비공개` 토글
- 좋아요 수 표시
- 공개곡인 경우 Explore 카드와의 연결성 유지

## 데이터 구조

### Music 확장

- `isPublic Boolean @default(true)`
- `artistHandle String?`
- `likeCount Int @default(0)` 또는 집계 계산

`artistHandle`은 나중에 별도 프로필 구조를 만들기 전까지 닉네임 기반 공개 페이지 주소로 쓸 수 있다.

### MusicLike

별도 좋아요 테이블 추가

- `id`
- `musicId`
- `userId`
- `createdAt`

제약:

- `(musicId, userId)` unique

용도:

- 총 좋아요 수
- 주간 좋아요 순위
- 월간 좋아요 순위

## 랭킹 집계 기준

### 주간 순위

- 최근 7일 기준 좋아요 수

### 월간 순위

- 최근 30일 기준 좋아요 수

초기 1차에서는 실시간 집계 쿼리로 시작하고, 데이터가 커지면 별도 집계 테이블 또는 배치 작업으로 전환한다.

## 사용자 경험 정책

### 공개곡 카드

- 제목
- 아티스트 닉네임
- 좋아요 수
- 플레이 버튼

추가 정책:

- 비로그인 사용자가 좋아요 버튼을 누르면 로그인 유도
- 미리듣기는 공개곡 preview 흐름만 허용

### 비공개 곡

- Explore에 노출하지 않음
- 아티스트 페이지에도 노출하지 않음
- 본인 `My Assets`에만 노출

## 구현 순서 제안

### 1단계

- `Music.isPublic` 추가
- `MusicLike` 모델 추가
- 기존 곡 기본값 `public` 반영

### 2단계

- `Explore` 페이지 1차 구현
- 공개곡 최신순 목록
- 카드형 UI
- 좋아요 버튼

### 3단계

- `Artists` 페이지 1차 구현
- 닉네임 클릭 흐름
- 공개곡만 노출

### 4단계

- 홈 화면 하단에 주간/월간/최신 공개곡 섹션 추가

### 5단계

- `Studio / Explore` 트리 메뉴 구조 정리

## 1차 구현에서 제외해도 되는 것

- 댓글
- 팔로우
- 플레이리스트
- 공개곡 상세 통계
- 프로필 이미지 업로드

## 결론

다음 실제 사용자 기능 확장은 아래 방향으로 간다.

- 곡 기본 공개
- 비로그인 청취 허용
- 공개곡 Explore 페이지
- 좋아요 구조
- 아티스트 공개곡 페이지
- 홈 하단 주간/월간 순위 섹션

이 구조가 잡히면 SongsAI Music은 `Create + My Assets` 중심 툴에서 `공개 감상 + 랭킹 + 아티스트 탐색`이 가능한 서비스로 확장될 수 있다.

## 구현 반영 메모 (2026-04-14)

설계 기준으로 1차 구현이 반영된 항목은 아래와 같다.

### 반영 완료

- `Music.isPublic` 추가, 기본값 `true`
- `MusicLike` 모델 추가
- 공개곡 목록 API `/api/v1/explore`
- 아티스트 공개곡 API `/api/v1/explore/artists/[artistId]`
- 좋아요 API `/api/v1/music/[id]/like`
- 프론트 `/explore` 페이지
- 프론트 `/artists/[artistId]` 페이지
- 헤더 메뉴 `Studio / Explore` 드롭다운 정리

### 아직 남은 항목

- `My Assets`의 공개/비공개 토글
- 홈 하단의 주간/월간/최신 공개곡 섹션
- Explore의 아티스트 목록 전용 화면
- 공개곡 카드의 추가 메타 정보/랭킹 표현 개선
- 주간/월간 집계 성능 최적화(필요 시 별도 집계 테이블 또는 배치)

### 새 세션에서 바로 이어갈 핵심 포인트

- 현재 공개곡은 기본 `public` 정책으로 동작한다.
- 비로그인 사용자도 공개곡 탐색과 미리듣기는 가능하다.
- 좋아요는 로그인한 사용자만 가능하다.
- Explore는 듣기 전용이며 다운로드/비디오 생성은 제공하지 않는다.
- 다음 구현 1순위는 `My Assets` 공개/비공개 토글이다.
