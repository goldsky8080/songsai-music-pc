# 현재 작업상태 및 다음순서

기준일: 2026-04-13

## 이번에 실제로 반영한 내용

### 1. Create 화면 정리

- `/create`를 실제 생성 화면 기준으로 정리했다.
- 상단 메뉴 `Create` 진입과 로그인 체크 흐름을 맞췄다.
- 버튼명은 `Create`로 통일했다.
- 모델 표시는 `Blue / Red / Gold`로 정리했다.
- 테스트용 보조 버튼과 불필요한 우측 요약 사이드바는 제거했다.

### 2. Create 생성 흐름 변경

- 프론트가 `POST /api/v1/music`를 1회 호출하는 구조로 정리했다.
- 백엔드는 provider 직접 호출 기반으로 바뀌었다.
- 생성 직후 최근 요청 카드가 바로 보이도록 연결했다.

### 3. 최근 요청 카드 UI 정리

- 최근 요청은 카드형으로 최근 6개 묶음까지만 보이게 정리했다.
- Suno가 한 번에 2곡을 주는 경우 `requestGroupId` 기준으로 한 카드에 묶었다.
- 카드 안에서 슬라이드로 곡을 넘겨볼 수 있게 했다.
- 상태 배지, 슬라이드 컨트롤, 카드 레이아웃을 현재 테마에 맞게 다듬었다.

### 4. 카드 미리듣기 방식 변경

- 카드 하단 기본 오디오 플레이어를 제거했다.
- 이미지 중앙 플레이 버튼으로 현재 슬라이드 곡을 재생하도록 변경했다.
- 슬라이드를 넘기면 현재 보이는 곡 기준으로 재생 대상이 바뀐다.
- 상태 배지와 슬라이드 화살표가 겹치지 않도록 위치를 조정했다.

### 5. 생성 직후 자동 반영 개선

- 예전에는 새로고침해야 제목, 커버, preview URL이 반영되는 경우가 있었다.
- 이를 줄이기 위해 최근 목록 조회 시 최근 5분 안의 곡에서 `title/mp3Url/imageUrl`이 비어 있으면 provider 상태를 가볍게 다시 확인하도록 바꿨다.
- 프론트에서도 제한적인 자동 갱신을 붙여 새로고침 없이 반영되도록 맞췄다.

### 6. 5분 전 / 5분 후 정책 정리

- 5분 전:
  - 미리듣기만 허용
  - 다운로드 / 비디오 생성 비활성화
  - raw preview URL 우선 사용
  - 파일 캐시와 완료 처리 금지

- 5분 후:
  - 듣기 / 다운로드 / 비디오 생성 시 provider 상태를 다시 확인
  - 최신 `title/mp3Url/imageUrl/tags/duration/timed lyrics` 반영
  - 필요한 파일을 서버에 저장
  - 최종 완료 처리 가능

### 7. 429 응답 확인 완료

- Suno 대기열 포화 시 응답 패턴을 실제로 확인했다.
- 응답은 `429 Too Many Requests`
- 본문에는 `Please wait for your other songs to finish generating, then try again.`와 `running_clip_ids`가 들어온다.
- 현재 판단은 내부 큐를 길게 쌓기보다 `429 안내 + 미차감` 방향이 더 맞다.

### 8. MusicAsset 활용 단계 진입

- `MusicAsset` 컬럼 확장은 이미 되어 있는 상태를 확인했다.
- 이번 단계부터는 설계가 아니라 실제 활용 단계로 들어왔다.
- 현재 기준:
  - 커버 이미지 캐시
  - 타임라인 가사 메타데이터 캐시
  - mp3 lazy cache
  흐름을 실제 요청 시점과 연결 중이다.

### 9. mp3 저장 로직 연결

- mp3는 생성 직후 즉시 저장하지 않는다.
- 다운로드나 비디오 생성 같은 후속 시점에 lazy cache로 저장하는 방향을 유지한다.
- 저장 위치는 `D:\wrapper\suno-api\storage\music-assets\audio` 기준이다.

## 현재 기준으로 이해한 동작

### 생성 직후

- provider 직접 호출
- 초기 `providerTaskId`, `title`, `mp3Url`, `imageUrl` 저장
- 최근 요청 카드에 즉시 반영

### 5분 전

- 미리듣기만 가능
- 다운로드 / 비디오 생성 비활성화
- preview 메타데이터만 lightweight refresh 가능
- 파일 저장과 완료 처리 금지

### 5분 후

- 첫 듣기/다운로드/비디오 생성 시점에 full sync
- 최신 제목/커버/mp3/tags/duration/timed lyrics 갱신
- 필요한 자산 캐시
- 완료 처리

## 아직 남아 있는 확인 포인트

### 1. 생성 직후 preview 안정성

- 모든 곡이 생성 직후 바로 preview 가능한지
- Suno 초기 응답에 `mp3Url`이 없는 케이스에서도 lightweight refresh로 충분히 붙는지 계속 확인 필요

### 2. 제목 갱신 최종 확인

- 생성 직후 제목이 없을 때
- 5분 후 full sync 시점에 실제 Suno 제목으로 정확히 바뀌는지 확인 필요

### 3. 2곡 묶음 카드 UX 미세조정

- 슬라이드 전환감
- 플레이 버튼 상태 표시
- 모바일 레이아웃
- 카드 정보 밀도

### 4. My Assets 최종 구조

- 현재는 최근 요청 카드 중심이다.
- 최종적으로는 2트랙 묶음 자산 관리 화면으로 확장 정리가 필요하다.
