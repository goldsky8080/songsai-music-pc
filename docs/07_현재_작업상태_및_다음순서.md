# 현재 작업상태 및 다음순서

기준일: 2026-04-16

## 이번 세션에서 실제 반영된 내용

### 1. 홈 / Create / My Assets / Explore 프론트 구조 정리

- 홈 화면을 React 기준으로 유지하면서 기존 테마를 최대한 보존하는 방향으로 정리했다.
- 하단 공개곡 섹션은 `주간 좋아요 순위 / 월간 좋아요 순위 / 최신 공개곡` 3개 컬럼 구조로 정리했다.
- 홈 메인 작업 패널의 최근 생성곡은 `GET /api/v1/music/recent?limit=10` 기준으로 다시 연결했다.
- 최근 생성곡 카드는 클릭 시 상단 플레이어 선택만 되는 구조에서, 카드 클릭 즉시 자동 재생되도록 수정했다.
- `Create`와 `My Assets`의 카드 구조는 React 컴포넌트 기준으로 통일했다.

### 2. 공개곡 / 좋아요 / Explore 흐름 반영

- 백엔드에 공개곡, 좋아요, 아티스트 공개곡 API가 연결된 상태를 기준으로 프론트 Explore를 붙였다.
- 홈 하단 섹션은 `/api/v1/explore` 응답을 사용한다.
- 공개곡의 재생, 좋아요, 정렬 기준은 현재 운영 구조와 맞춘 상태다.

### 3. 다운로드 / 공개 미리듣기 경로 정리

- 운영 환경에서 음악 다운로드가 프론트 `/api/proxy/...`를 타다가 Cloudflare 502가 나던 문제를 정리했다.
- 프론트는 로컬에서는 `/api/proxy`, 운영에서는 `https://api.songsai.org` 직접 경로를 쓰도록 정리했다.
- 홈 공개 데이터 로딩은 브라우저 환경 이슈를 줄이기 위해 `pc.songsai.org/api/proxy/...` 우선, 실패 시 `https://api.songsai.org/...` 직접 재시도 구조로 보강했다.

### 4. 비디오 생성 / 비디오 다운로드 흐름 정리

- `My Assets`에서는 완료된 비디오가 `비디오 다운로드`로 바뀌도록 정리했다.
- `Create`에서도 동일하게 완료 후 `비디오 다운로드` 버튼으로 바뀌도록 수정했다.
- 카드 안에 비디오 생성 진행바를 표시하도록 정리했다.
- 진행바는 백엔드 퍼센트가 아니라 프론트 시간 기준으로 12%~95%까지 자연스럽게 올라가는 방식이다.
- 실제 완료 상태(`videoStatus=completed`, `canDownloadVideo=true`)가 오면 진행바는 숨기고 `비디오 다운로드`만 남긴다.
- `비디오 생성` 버튼을 누른 직후 바로 진행 상태가 보이도록 로컬 시작 시각을 기록하는 방식으로 바꿨다.

### 5. 비디오 렌더링 속도 개선

- 백엔드에서 `zoompan`을 제거해 정적 커버 이미지 기반 비디오로 바꿨다.
- 비디오 해상도를 `1080x1920`에서 `720x1280`으로 낮췄다.
- 프레임을 `30fps`에서 `24fps`로 낮췄다.
- 인코딩 preset을 더 빠른 쪽으로 조정했다.
- 비디오 워커 동시 처리 수를 `1`에서 `4`로 늘렸다.

### 6. 신규 서버 배포 / 운영 반영 중 확인된 실제 문제

- 프론트와 백엔드 저장소 remote가 서로 달라서, 백엔드 수정이 잘못된 remote로만 push된 적이 있었다.
- 운영 서버의 백엔드 `~/services/wrapper`는 `songsai-api.git` 기준으로 pull된다는 점을 확인했다.
- 프론트 Next.js production build가 없어서 `.next/prerender-manifest.json`, `.next/BUILD_ID` 관련 오류가 났고, `npm run build` 후 `pm2 restart`로 해결했다.
- 백엔드 Prisma 스키마와 운영 DB 컬럼(`Music.tags`) 불일치가 있었고, DB 상태 확인 후 정리했다.
- 홈 최근 생성곡은 내부 `3100` 호출과 공용 `https://api.songsai.org` 호출 둘 다 정상 응답하는 것까지 확인했다.
- 다만 브라우저 환경에서 `/api/proxy`와 직접 API 호출이 상황에 따라 흔들려서, 홈 로더를 이중 fallback 구조로 보강했다.

## 현재 기준으로 아직 점검이 더 필요한 것

### 1. 홈 메인 작업 패널의 운영 브라우저 안정성

- 서버 내부 `3100` direct 호출은 정상이다.
- 공용 `https://api.songsai.org/api/v1/music/recent` direct 호출도 정상이다.
- 그러나 일부 브라우저 세션에서 홈 공개 feed fetch가 502로 흔들린 적이 있어, 운영 브라우저에서 최종 재검증이 필요하다.

### 2. My Assets 공개/비공개 토글

- 토글 실패 시 메시지는 더 명확하게 정리했다.
- 실제 운영 세션에서 401인지, API 오류인지 재확인이 필요하다.

### 3. 비디오 생성 상태 UX 미세조정

- 프론트 진행바와 실제 완료 전환은 구현되었다.
- 카드 버튼 문구와 disabled 상태가 모든 케이스에서 자연스러운지 추가 점검이 필요하다.
