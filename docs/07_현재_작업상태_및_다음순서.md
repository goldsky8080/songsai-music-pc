### 2. 로컬 개발 구조 안정화

- 프론트: `http://localhost:3000`
- 백엔드: `http://localhost:3100`
- 프론트는 직접 `3100`을 때리지 않고 `/api/proxy/...`를 통해 백엔드에 연결하는 구조로 정리했다.
- 이 구조로 로컬 로그인/세션/CORS 문제를 줄였다.

### 3. songsai-api 기본 기능 연결

- `POST /api/v1/music`
  - 음악 생성 요청 저장
- `GET /api/v1/music`
  - 내 음악 목록 조회
- `GET /api/v1/music/recent`
  - 완료곡 최근 목록 조회
- `POST /api/v1/jobs/run`
  - 수동 worker 실행

### 4. 큐 기반 음악 생성 흐름 검증

- 음악 생성 요청이 `Music`, `GenerationJob`에 저장되는 것 확인
- worker 실행 후 generation job 처리 확인
- 5분 후 poll job 실행 시 완료곡 상태 전환 확인
- 완료곡이 홈 최근 목록과 자산 목록에 반영되는 것 확인

### 5. 다운로드 흐름 정리

- 오디오 다운로드는 새 탭이 아니라 attachment 응답으로 바로 다운로드되도록 변경
- 파일명은 제목 기반으로 내려가도록 정리
- 다운로드 시점에 provider 상태를 다시 확인해 최신 URL을 반영하는 방향으로 정리

### 6. timed lyrics 수집 확인

- provider에서 시간 정보가 포함된 aligned lyrics 응답이 오는 것 확인
- 이 데이터는 이후 비디오 렌더 입력으로 사용할 계획

### 7. My Assets 화면 1차 정리

- 기존 템플릿의 겹치는 섹션 일부를 제거했다
- 푸터 레이아웃이 하단으로 가도록 다시 정리했다
- 상단 메뉴는 `Home / Create / My Assets / Pricing / Support` 기준으로 맞추는 방향으로 정리 중이다

### 8. 커버 이미지 캐시 1차 시도

- 완료곡 커버를 우리 서버 쪽으로 캐시하려는 구조를 붙였다
- 하지만 캐시가 실제로 없는 상태에서 completed 곡을 무조건 서버 cover route로 바꾸는 문제를 확인했다
- 이 문제는 다시 되돌려서, 현재는 캐시 전에는 기존 `music.imageUrl`을 그대로 사용하도록 수정했다

### 9. 서버 운영 쪽 정리

- 신규 서버 재부팅 후 점검 순서 문서를 만들었다
- `pm2 startup` / `pm2 save` 설정까지 마쳐서 재부팅 후 자동 복구 가능성을 높였다
- nginx / pm2 / 외부 도메인 확인 절차를 문서화했다

## 현재 합의된 큰 방향

### 백엔드 / 프론트 역할

- `songsai-api`가 진짜 서비스 백엔드 원본
- `songsai-music-pc`는 API 호출 중심의 얇은 프론트
- `D:\music`은 참고용 구조/도메인 로직 소스

### 홈과 자산 화면

- 홈은 완료된 곡만 보여준다
- `My Assets`는 내가 생성한 곡을 보여주고, 진행 중인 곡도 포함한다

### 다운로드 / 자산 정책

- 다운로드는 새 탭 대신 바로 attachment 다운로드
- 커버 이미지는 우리 서버 캐시가 있으면 우리 URL 사용
- 캐시가 없으면 기존 provider 이미지 URL 사용
- mp3는 lazy 캐시 허용

### 비디오 정책

- 비디오는 provider `videoUrl` 의존 구조로 가지 않는다
- 우리 서버에서 `mp3 + cover image + timed lyrics`를 재료로 생성한다
- 비디오 생성도 `GenerationJob` 큐로 처리한다

### 스토리지 정책

- SSD 1TB: hot storage
  - `/srv/media/music/...`
  - `/srv/media/image/...`
  - `/srv/media/video/...`
  - `/srv/media/tmp/...`
- NAS 4TB: archive storage
  - `/srv/oldmedia/music/...`
  - `/srv/oldmedia/image/...`
  - `/srv/oldmedia/video/...`
  - `/srv/oldmedia/tmp/...`
- HDD 500GB는 우선 보조/백업
- 앱 코드는 `createdAt`으로 직접 경로를 나누지 않고, 향후 `MusicAsset`의 저장 메타데이터로 관리한다

## 현재 남아 있는 문제

### 1. `vendor-scripts.tsx` 불안정성

- 템플릿 DOM을 후처리로 많이 덮어쓰는 구조라 안정성이 떨어진다
- 문자열 인코딩 깨짐 흔적도 일부 남아 있다
- 장기적으로는 이 구조를 줄여야 한다

### 2. `My Assets` 구조가 아직 최종 아님

- 현재는 트랙 단위 카드처럼 보인다
- 하지만 실제 생성 1회는 보통 2곡이 오므로, 최종적으로는 2트랙 묶음 구조가 필요하다

### 3. 커버 이미지 캐시 시점이 아직 미완성

- 현재는 캐시 전에는 provider 이미지 URL을 쓰도록 되돌린 상태
- 캐시 성공 시점에만 우리 서버 URL로 전환하는 정확한 로직은 다음 단계에서 정리해야 한다

### 4. `VIDEO_RENDER`는 아직 최종 구조가 아님

- 지금은 임시적인 흐름이 섞여 있다
- 최종적으로는 ffmpeg 기반 서버 렌더 큐로 바꿔야 한다
