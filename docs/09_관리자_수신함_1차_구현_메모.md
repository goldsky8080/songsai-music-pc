# 관리자 수신함 1차 구현 메모

기준일: 2026-04-14

## 이번에 반영한 내용

### 1. 문의 메일 webhook 저장

- 백엔드에 문의 메일 수신 webhook 엔드포인트를 연결했다.
- 엔드포인트:
  - `POST /api/v1/inbound-email/webhook`
- `INBOUND_WEBHOOK_SECRET`가 설정돼 있으면 헤더 또는 쿼리 secret으로 보호한다.
- 수신 메일은 `InboundEmail` 테이블에 저장한다.

### 2. 관리자 수신함 API

- 관리자 전용 목록 API:
  - `GET /api/v1/admin/inbound-emails`
- 관리자 전용 상세 API:
  - `GET /api/v1/admin/inbound-emails/:id`
- 상태 변경 API:
  - `PATCH /api/v1/admin/inbound-emails/:id`

현재 상태 값:
- `NEW`
- `READ`
- `REPLIED`
- `ARCHIVED`

### 3. 첨부파일 저장 1차

- `InboundEmailAttachment` 모델을 추가했다.
- webhook payload 안에 첨부파일 데이터가 있으면 로컬 스토리지에 저장한다.
- 현재 저장 위치:
  - `D:\wrapper\suno-api\storage\inbound-email-attachments`
- base64 또는 텍스트 기반 첨부 데이터는 실제 파일로 저장한다.
- 파일 데이터가 없더라도 메타데이터는 DB에 남긴다.

### 4. 관리자 수신함 화면

- 프론트에 `/admin/inbox` 화면을 추가했다.
- 접근 권한:
  - 비로그인: `/login?next=/admin/inbox`
  - 관리자 아님: `/`
  - `ADMIN`만 접근 가능
- 화면 구성:
  - 왼쪽: 메일 목록
  - 오른쪽: 메일 상세
  - 상태 변경 버튼: 읽음 / 답변 완료 / 보관
- `NEW` 메일을 열면 자동으로 `READ`로 바뀐다.

### 5. 관리자 공통 네비

- 관리자 화면 내부에 공통 네비를 추가했다.
- 현재 메뉴:
  - `Members`
  - `Inbox`
- 상단 메인 헤더에는 넣지 않고, 관리자 화면 내부에서만 보이게 했다.

## 같이 정리된 기준

- `admin/users` API도 `ADMIN`만 허용하도록 정리했다.
- 관리자 화면은 로그인만으로는 접근되지 않고, 관리자 권한까지 있어야 한다.

## 남은 작업

1. 실제 Resend inbound payload 기준 첨부 파싱 케이스 보강
2. 첨부파일 다운로드 엔드포인트 추가
3. 수신 메일 답변 메모 / 처리 이력 / 담당자 지정 기능 확장
4. 필요 시 관리자 대시보드 홈 추가
## 현재 판단

- 관리자 영역은 여기서 1차 구현으로 보류한다.
- 이후 우선순위는 실제 사용자 메뉴 쪽 화면 정리로 이동한다.
