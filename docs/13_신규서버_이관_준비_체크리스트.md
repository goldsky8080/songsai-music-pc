# 신규 서버 이관 준비 체크리스트

기준일: 2026-04-15

이 문서는 `songsai-music-pc` 프론트와 `songsai-api` 백엔드를 신규 서버로 옮길 때
바로 따라갈 수 있는 실제 준비 순서를 정리합니다.

## 1. 이번 이관에서 먼저 확인할 것

- 프론트 저장소: `D:\songsai-music-pc`
- 백엔드 저장소: `D:\wrapper\suno-api`
- 로컬 개발 포트
  - 프론트: `3000`
  - 백엔드: `3100`
- 프론트는 브라우저에서 백엔드 포트를 직접 치지 않고 `/api/proxy/...`로 붙는다.
- 서버 이관도 프론트와 백엔드를 따로 올리되, 최종 연결은 프론트 환경값으로 맞춘다.

## 2. 배포 전 로컬에서 끝내둘 것

- `songsai-music-pc` 최신 작업 커밋
- `songsai-api` 최신 작업 커밋
- 프론트 타입체크 통과
  - `cd D:\songsai-music-pc`
  - `npx tsc --noEmit`
- 백엔드 Prisma / 타입체크 확인
  - `cd D:\wrapper\suno-api`
  - `npx prisma generate`
  - `npx tsc --noEmit`

## 3. 이번 시점에 서버 반영 대상

  - 예시: `~/services/songsai-music-pc`
  - 예시: `~/services/suno-api`

## 5. 서버 환경값 준비

### 프론트 서버 env

프론트는 서버에서 백엔드 운영 주소를 바라봐야 합니다.

예시:

```env
NEXT_PUBLIC_SONGSAI_API_URL=https://api.songsai.org
```

### 백엔드 서버 env

최소 확인 항목:

```env
APP_URL=https://api.songsai.org
FRONTEND_URL=https://pc.songsai.org
DATABASE_URL=...
AUTH_SECRET=...
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
RESEND_API_KEY=...
```

주의:

- `.env.local`을 서버에 그대로 복사하지 않는다.
- 서버별 `.env`는 별도 작성한다.
- 운영 서버 env 변경 후에는 `pm2 restart --update-env` 기준으로 반영한다.

## 6. 서버 반영 순서

1. 신규 서버에 백엔드 저장소 배치
2. 백엔드 `.env` 작성
3. `npm install`
4. `npx prisma generate`
5. 필요 시 `npx prisma migrate deploy`
6. 백엔드 PM2 등록
7. 백엔드 포트 응답 확인
8. 프론트 저장소 배치
9. 프론트 `.env` 작성
10. `npm install`
11. `npm run build`
12. 프론트 PM2 등록
13. nginx 연결
14. HTTPS 연결
15. 브라우저 실검증

## 7. PM2 / nginx 기준 파일

프론트 저장소 안에 이미 배포 기준 파일이 있습니다.

- `deploy/pm2.pc.songsai-music.config.cjs`
- `deploy/nginx.pc.songsai.org.conf.example`

프론트 예시:

```bash
cd ~/services/songsai-music-pc
npm install
npm run build
pm2 start deploy/pm2.pc.songsai-music.config.cjs
pm2 save
```

nginx 예시:

```bash
sudo cp deploy/nginx.pc.songsai.org.conf.example /etc/nginx/sites-available/pc.songsai.org
sudo ln -s /etc/nginx/sites-available/pc.songsai.org /etc/nginx/sites-enabled/pc.songsai.org
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 이번 기준에서 특히 놓치면 안 되는 것

- 프론트와 백엔드 저장소가 분리되어 있으니 둘 다 커밋해야 한다.
- 프론트는 `3000`, 백엔드는 `3100` 로컬 기준이지만 운영 포트는 PM2 / nginx 기준으로 따로 본다.
- 공개곡 토글은 프론트만 올리면 안 되고 백엔드 visibility API가 같이 올라가야 한다.
- 홈과 다른 메뉴의 상단 헤더는 이제 공통 구조이므로, 배포 후에도 동일하게 보이는지 꼭 확인한다.
