# 신규서버 이관 준비 체크리스트

기준일: 2026-04-16

이 문서는 `songsai-music-pc` 프론트와 `songsai-api` 백엔드를 신규 서버에 반영할 때 필요한 준비물과 실제 배포 순서를 정리한 운영 기준 문서다.

## 1. 저장소 / 경로 / 포트 기준

### 로컬 작업 경로

- 프론트: `D:\songsai-music-pc`
- 백엔드: `D:\wrapper\suno-api`

### 신규 서버 운영 경로

- 프론트: `~/services/songsai-music-pc`
- 백엔드: `~/services/wrapper`

### 포트 기준

- 프론트: `3000`
- 백엔드: `3100`

## 2. 제일 중요한 remote 기준

### 프론트

- 운영 서버 프론트는 `songsai-music-pc` 저장소 `origin/main`을 pull한다.

### 백엔드

- 운영 서버 백엔드 `~/services/wrapper`는 `songsai-api.git` 기준으로 pull한다.
- 로컬 백엔드 저장소에는 remote가 둘 이상 있을 수 있으므로, 실제 운영 배포용 remote가 어디인지 반드시 확인해야 한다.

실제 확인 명령:

```bash
git remote -v
git branch --show-current
```

운영 서버 `~/services/wrapper`에서 `git remote -v` 결과가 `songsai-api.git`를 가리키는지 반드시 확인한다.

## 3. 서버 기본 패키지

신규 서버 1회 준비:

```bash
sudo apt update
sudo apt install -y nginx ffmpeg postgresql-client
sudo npm install -g pm2
node -v
npm -v
pm2 -v
ffmpeg -version
```

## 4. 환경변수 기준

### 프론트 `~/services/songsai-music-pc/.env`

운영 기준 핵심 값:

```env
NEXT_PUBLIC_SONGSAI_API_URL=https://api.songsai.org
SONGSAI_API_BASE_URL=https://api.songsai.org
```

설명:

- `NEXT_PUBLIC_SONGSAI_API_URL`: 브라우저에서 백엔드 API를 붙을 때 기준
- `SONGSAI_API_BASE_URL`: 공개 미리듣기, 프록시 fallback 등 서버 측 기준 URL

### 백엔드 `~/services/wrapper/.env`

운영 기준 핵심 값:

```env
DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/DBNAME?schema=songsai_api
APP_URL=https://api.songsai.org
FRONTEND_URL=https://pc.songsai.org
FRONTEND_URLS=https://pc.songsai.org
AUTH_SECRET=...
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
RESEND_API_KEY=...
```

주의:

- 운영에서는 `schema=songsai_api` 기준 DB를 사용한다.
- `RESEND_API_KEY`가 없으면 이메일 회원가입 인증 메일이 동작하지 않는다.
- env를 수정한 뒤에는 항상 `pm2 restart ... --update-env`로 반영한다.

## 5. 백엔드 배포 순서

```bash
cd ~/services/wrapper
git pull origin main
git log -1 --oneline
npm install
npx prisma generate
npm run build
pm2 restart songsai-api --update-env
pm2 save
pm2 flush songsai-api
sleep 3
curl -I http://127.0.0.1:3100
```

### 백엔드 배포 후 필수 확인

```bash
curl http://127.0.0.1:3100/api/v1/music/recent?limit=10
curl 'http://127.0.0.1:3100/api/v1/explore?sort=latest&limit=6&offset=0'
curl 'http://127.0.0.1:3100/api/v1/explore?sort=weekly&limit=6&offset=0'
curl 'http://127.0.0.1:3100/api/v1/explore?sort=monthly&limit=6&offset=0'
```

```bash

curl -v http://127.0.0.1:3000/api/proxy/api/v1/explore?sort=latest&limit=6&offset=0
```

## 7. nginx / 도메인 기준

### 프론트 도메인

- `pc.songsai.org` -> `127.0.0.1:3000`

### 백엔드 도메인

- `api.songsai.org` -> `127.0.0.1:3100`

### 프론트 nginx 예시

`deploy/nginx.pc.songsai.org.conf.example` 기준으로 적용한다.

### 백엔드 nginx 확인 포인트

- `api.songsai.org`가 실제로 `127.0.0.1:3100`으로 proxy_pass 되는지 확인
- Cloudflare를 쓰더라도 내부 upstream은 `127.0.0.1:3100`이어야 함

## 8. 자주 겪은 실제 장애와 해결 포인트

### 1. 프론트 `.next` production build 없음

오류 예시:

- `.next/prerender-manifest.json` 없음
- `.next/BUILD_ID` 없음
- `Could not find a production build in the '.next' directory`

해결:

```bash
cd ~/services/songsai-music-pc
npm run build
pm2 restart songsai-music-pc --update-env
```

### 2. 백엔드 Prisma 컬럼 불일치

오류 예시:

- `The column Music.tags does not exist`

해결 방향:

- 운영 DB 스키마 확인
- Prisma schema와 DB 스키마 맞춤
- `npx prisma generate`
- 필요시 DB 스키마 정합성 재검증

### 3. 잘못된 remote로 push

문제:

- 로컬 백엔드 수정을 `suno-api.git`에만 push하고, 운영 서버는 `songsai-api.git`를 보고 있어서 pull해도 최신 반영이 안 됨

해결:

- 운영 서버가 보는 remote를 기준으로 push해야 함
- 커밋 반영 후 운영 서버에서 `git log -1 --oneline`으로 커밋 hash까지 확인

### 4. 홈 공개 feed 502

문제:

- 홈 메인 작업 패널과 공개곡 섹션 fetch가 브라우저에서 502로 흔들림

실제 원인:

- 프론트 `/api/proxy/...`가 운영에서도 외부 `https://api.songsai.org`를 다시 타는 구조라서, 프록시가 내부 백엔드가 아니라 외부 도메인을 한번 더 경유함

해결:

- 운영 프론트의 `/api/proxy`는 내부 `127.0.0.1:3100`을 기본 upstream으로 사용
- 홈 로더는 `/api/proxy` 실패 시 직접 `https://api.songsai.org` fallback 재시도

### 5. 다운로드 502

문제:

- 음악 다운로드가 프론트 도메인을 통해 잘못 프록시되면서 Cloudflare 502 발생

해결:

- 다운로드 / 비디오 다운로드는 운영에서 `https://api.songsai.org/...` 직접 경로 사용

## 9. 운영 검증 체크리스트

### 프론트

- `/` 접속
- 홈 메인 작업 패널 최근 생성곡 노출
- 카드 클릭 즉시 재생
- 하단 `주간 / 월간 / 최신 공개곡` 노출
- `Create` 진입
- `My Assets` 진입
- 비디오 생성 진행바 노출
- 완료 후 `비디오 다운로드` 전환

### 백엔드

- `/api/v1/music/recent`
- `/api/v1/explore`
- `/api/v1/music`
- `/api/v1/music/[id]/download`
- `/api/v1/music/[id]/video/download`

### 인증

- 로그인
- 회원가입
- 이메일 인증 메일 발송 여부
- `My Assets` 공개/비공개 토글

```bash
tail -n 100 ~/.pm2/logs/songsai-api-error.log
tail -n 100 ~/.pm2/logs/songsai-api-out.log
```

```bash
pm2 logs songsai-music-pc --lines 100
tail -n 100 ~/.pm2/logs/songsai-music-pc-error.log
tail -n 100 ~/.pm2/logs/songsai-music-pc-out.log
```

## 11. 이관 직전 마지막 확인

- 프론트 `git status` 깨끗한지
- 백엔드 `git status` 깨끗한지
- 운영 서버가 pull할 remote가 맞는지
- 프론트 `.env`, 백엔드 `.env` 운영값이 맞는지
- `ffmpeg -version` 되는지
- `pm2 status`에서 `songsai-api`, `songsai-music-pc` 둘 다 online인지
