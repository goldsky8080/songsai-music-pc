## 3. 실제로 겪은 문제

### 문제 1. 프론트 production build 없음

오류:

- `.next/prerender-manifest.json` 없음
- `.next/BUILD_ID` 없음
- `Could not find a production build in the '.next' directory`

원인:

- `pm2 restart`만 하고 `npm run build`를 안 했거나, 빌드 실패 상태에서 PM2만 재시작함

해결:

```bash
cd ~/services/songsai-music-pc
npm run build
pm2 restart songsai-music-pc --update-env
pm2 save
```

### 문제 2. Explore 빌드 에러

오류:

- `useSearchParams() should be wrapped in a suspense boundary at page "/explore"`

원인:

- Explore 페이지가 production build 기준 Suspense 요건을 만족하지 못함

해결:

- 프론트에서 Explore production build 수정 후 커밋/푸시
- 서버에서 `git pull`, `npm run build` 재실행

### 문제 3. 백엔드 DB 컬럼 불일치

오류:

- `The column Music.tags does not exist`

원인:

- 운영 DB 스키마와 Prisma 스키마가 일시적으로 맞지 않음

해결 과정:

- `psql`로 `songsai_api."Music"` 테이블 구조 직접 확인
- `tags` 컬럼 존재 여부 검증
- PM2 로그 flush 후 재확인
- 최신 build / restart 이후 recent / explore API 재검증

결론:

- 최종적으로 `recent`, `explore` API는 정상 200과 JSON 응답을 반환하는 상태까지 확인했다.

### 문제 4. 백엔드 remote 혼선

원인:

- 로컬 백엔드 저장소에는 remote가 둘 이상 있었고, 운영 서버는 `songsai-api.git`를 보고 있었음
- 일부 커밋을 잘못된 remote에만 push해서 서버가 pull해도 최신이 안 내려왔음

해결:

- 운영 서버가 참조하는 remote 기준으로 다시 push
- 서버에서 `git log -1 --oneline`으로 커밋 hash를 직접 확인

### 문제 5. 홈 recent / explore 502

현상:

- 브라우저 devtools에서 `recent?limit=10`, `explore?...` 요청이 502로 뜬 적이 있음
- 그러나 서버 내부 `curl http://127.0.0.1:3100/api/v1/music/recent?limit=10`은 정상
- 공용 `curl https://api.songsai.org/api/v1/music/recent?limit=10`도 정상

결론:

- 백엔드 route 자체 문제는 아니었음
- 프론트 `/api/proxy`의 upstream 기준과 브라우저 fetch 경로가 섞여 있던 문제였음

해결:

1. 운영 프록시 route는 내부 `127.0.0.1:3100` 기준으로 정리
2. 홈 로더는 `/api/proxy` 우선, 실패 시 직접 `https://api.songsai.org` fallback

### 문제 6. 다운로드 502

현상:

- 음악 다운로드 시 Cloudflare 502

원인:

- 프론트가 운영에서도 `/api/proxy/.../download`를 잘못 타던 구조

해결:

- 다운로드와 비디오 다운로드는 운영에서 `https://api.songsai.org/...` 직접 경로로 전환

### 문제 7. 비디오 생성은 되었는데 버튼이 안 바뀜

현상:

- DB와 파일은 생성되어 있음
- `GenerationJob`은 `COMPLETED`
- `Video.status`도 `COMPLETED`
- `Music.videoUrl`도 들어가 있음
- 그런데 프론트 `Create` / `My Assets` 카드가 `비디오 다운로드`로 안 바뀌는 경우가 있었음

원인:

- 프론트가 `videoId`, `videoStatus`, `canDownloadVideo`를 보는 조건과 화면 버튼 전환 조건이 일부 화면에서 서로 달랐음

해결:

- `My Assets`, `Create` 둘 다 동일한 완료 판정 기준으로 정리

### 문제 8. 비디오 렌더링 속도

초기 상태:

- `zoompan`
- `1080x1920`
- `30fps`
- 동시 처리 1

개선 후:

- 정적 커버 이미지
- `720x1280`
- `24fps`
- 동시 처리 4

## 4. 실제 서버 검증에 쓴 명령

```bash

curl http://127.0.0.1:3100/api/v1/music/recent?limit=10
curl 'http://127.0.0.1:3100/api/v1/explore?sort=latest&limit=6&offset=0'
curl 'http://127.0.0.1:3100/api/v1/explore?sort=weekly&limit=6&offset=0'
curl 'http://127.0.0.1:3100/api/v1/explore?sort=monthly&limit=6&offset=0'
```

```bash
curl -v https://api.songsai.org/api/v1/music/recent?limit=10
curl -v https://api.songsai.org/api/v1/explore?sort=latest&limit=6&offset=0
```

```bash
curl -I http://127.0.0.1:3000
curl -v http://127.0.0.1:3000/api/proxy/api/v1/music/recent?limit=10
```
