# 전체구조 개요

이 문서는 SONGSAI 관련 저장소 3개를 한 번에 이해하기 위한 문서입니다.

## 저장소 구성

### 1. 메인 서비스

- 경로: `D:\music`
- 역할:
  - 로그인과 사용자 세션
  - 크레딧과 입금 요청
  - 음악 생성 요청
  - 최근 곡 목록
  - 재생과 다운로드
  - 자산 관리
  - 비디오 생성
  - 관리자 화면 일부

관련 문서:

- [메인 안내](D:\music\README.md)
- [시스템 구조](D:\music\docs\시스템_구조.md)
- [운영·배포](D:\music\docs\운영_배포.md)
- [프로젝트 이력](D:\music\docs\프로젝트_이력.md)

### 2. 입금 자동화 보조 서비스

- 경로: `D:\songsai-bank-bridge`
- 역할:
  - SMS webhook 수신
  - 입금 문자 파싱
  - 메인 서비스의 입금 요청과 자동 매칭 보조
  - 모니터링 대시보드

현재 판단:

- 이 서비스는 메인 결제 시스템이 아니라 bank transfer 자동화 보조 역할에 집중하는 것이 맞다.

### 3. 음악 생성 wrapper

- 경로: `D:\wrapper\suno-api`
- 역할:
  - SONGS/Suno 쪽 호출
  - 세션, 쿠키, CAPTCHA 처리
  - 생성 요청과 결과 회수
  - aligned lyrics 같은 부가 처리

현재 판단:

- 가장 운영 리스크가 큰 계층이다.
- 메인 앱이 흔들리는 원인 중 많은 부분이 이 wrapper 안정성과 연결된다.

## 지금 구조의 특징

- `music`이 사용자 앱이면서 일부 비즈니스 로직을 많이 가지고 있다.
- `wrapper`는 실제 생성 엔진과 자동화 계층 역할을 한다.
- `bank-bridge`는 입금 자동화 보조 역할을 한다.

## 앞으로의 권장 방향

- `songsai-music`은 점점 얇은 프론트가 되어야 한다.
- `songsai-api`가 점점 메인 백엔드가 되어야 한다.
- `songsai-bank-bridge`는 specialized helper로 유지하는 것이 좋다.
